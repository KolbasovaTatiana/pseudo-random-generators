\chapter{Лабораторная работа №3. Исследование генераторов псевдослучайных чисел }
\hypertarget{index}{}\label{index}\index{Лабораторная работа №3. Исследование генераторов псевдослучайных чисел@{Лабораторная работа №3. Исследование генераторов псевдослучайных чисел}}
\label{index_md__r_e_a_d_m_e}%
\Hypertarget{index_md__r_e_a_d_m_e}%
\hypertarget{index_autotoc_md1}{}\doxysection{\texorpdfstring{Цель работы}{Цель работы}}\label{index_autotoc_md1}
Исследовать качество и скорость работы нескольких генераторов псевдослучайных чисел, сравнить их статистические характеристики и оценить пригодность для практического использования.\hypertarget{index_autotoc_md2}{}\doxysection{\texorpdfstring{Постановка задачи}{Постановка задачи}}\label{index_autotoc_md2}
В работе реализованы и исследованы следующие генераторы псевдослучайных чисел:\+


\begin{DoxyItemize}
\item {\ttfamily \doxylink{class_l_c_g___bays_durham}{LCG\+\_\+\+Bays\+Durham}} — линейный конгруэнтный генератор с перемешиванием Bays-\/\+Durham.
\item {\ttfamily \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT}} — генератор на основе схемы xorshift128 с дополнительным смешиванием.
\item {\ttfamily \doxylink{class_additive_l_f_g___rotate_xor}{Additive\+LFG\+\_\+\+Rotate\+Xor}} — аддитивный лаговый рекуррентный генератор с циклическими сдвигами и XOR.
\end{DoxyItemize}

Для каждого генератора требуется:\+


\begin{DoxyEnumerate}
\item Сформировать 20 выборок по 1000 чисел в диапазоне от 0 до 9999.
\item Вычислить описательные статистики:\+ среднее значение, стандартное отклонение, коэффициент вариации и статистику хи-\/квадрат.
\item Проверить выборки на равномерность распределения по критерию хи-\/квадрат.
\item Выполнить набор NIST-\/подобных тестов для битовых последовательностей.
\item Измерить скорость генерации различного количества случайных чисел.
\item Сохранить результаты в CSV-\/файлы для построения графиков и включения в документацию.
\end{DoxyEnumerate}\hypertarget{index_autotoc_md3}{}\doxysection{\texorpdfstring{Теоретические сведения}{Теоретические сведения}}\label{index_autotoc_md3}
Псевдослучайные числа формируются детерминированными алгоритмами, которые по начальному зерну порождают последовательность, внешне похожую на случайную. Качество генератора оценивается по статистическим свойствам последовательности и по отсутствию заметных закономерностей в распределении значений и битов.

Для анализа в работе используются следующие характеристики:\+


\begin{DoxyItemize}
\item Среднее значение выборки.
\item Стандартное отклонение.
\item Коэффициент вариации.
\item Статистика критерия хи-\/квадрат.
\item p-\/value для NIST-\/подобных тестов.
\end{DoxyItemize}

Если значение статистики хи-\/квадрат не превышает критического уровня, гипотеза о равномерности распределения не отвергается. Для NIST-\/подобных тестов чем выше p-\/value, тем меньше оснований считать последовательность неслучайной.\hypertarget{index_autotoc_md4}{}\doxysection{\texorpdfstring{Описание реализованных генераторов}{Описание реализованных генераторов}}\label{index_autotoc_md4}
\hypertarget{index_autotoc_md5}{}\doxysubsection{\texorpdfstring{LCG\+\_\+\+Bays\+Durham}{LCG\+\_\+\+Bays\+Durham}}\label{index_autotoc_md5}
Генератор основан на линейном конгруэнтном методе. Для уменьшения корреляции между соседними значениями используется таблица перемешивания Bays-\/\+Durham из 32 элементов. После выбора значения из таблицы результат дополнительно проходит через этап смешивания битов.\hypertarget{index_autotoc_md6}{}\doxysubsection{\texorpdfstring{Xorshift128\+\_\+\+MLT}{Xorshift128\+\_\+\+MLT}}\label{index_autotoc_md6}
Генератор использует схему {\ttfamily xorshift128}, основанную на последовательности операций XOR и сдвигов. После получения очередного значения выполняется дополнительное нелинейное смешивание, что улучшает статистические свойства выходной последовательности.\hypertarget{index_autotoc_md7}{}\doxysubsection{\texorpdfstring{Additive\+LFG\+\_\+\+Rotate\+Xor}{Additive\+LFG\+\_\+\+Rotate\+Xor}}\label{index_autotoc_md7}
Данный генератор использует аддитивную лаговую рекуррентную схему с кольцевым буфером. Для повышения качества последовательности в вычислениях применяются циклические сдвиги, XOR и дополнительная функция перемешивания.\hypertarget{index_autotoc_md8}{}\doxysubsection{\texorpdfstring{Standard\+MT}{Standard\+MT}}\label{index_autotoc_md8}
Для сравнения скорости работы дополнительно используется стандартный генератор {\ttfamily std::\+mt19937\+\_\+64}, представляющий собой реализацию алгоритма Mersenne Twister.\hypertarget{index_autotoc_md9}{}\doxysection{\texorpdfstring{Описание статистических тестов}{Описание статистических тестов}}\label{index_autotoc_md9}
\hypertarget{index_autotoc_md10}{}\doxysubsection{\texorpdfstring{Описательные статистики}{Описательные статистики}}\label{index_autotoc_md10}
Для каждой выборки вычисляются:\+


\begin{DoxyItemize}
\item среднее значение;
\item стандартное отклонение;
\item коэффициент вариации;
\item статистика хи-\/квадрат;
\item результат проверки на равномерность.
\end{DoxyItemize}\hypertarget{index_autotoc_md11}{}\doxysubsection{\texorpdfstring{NIST-\/подобные тесты}{NIST-\/подобные тесты}}\label{index_autotoc_md11}
В работе реализованы следующие тесты для битовой последовательности длиной 1000000 бит:\+


\begin{DoxyItemize}
\item Монобитный тест.
\item Тест частоты в блоках.
\item Тест серий.
\item Тест на длину наибольшей серии единиц.
\item Тест кумулятивных сумм в прямом направлении.
\item Тест кумулятивных сумм в обратном направлении.
\end{DoxyItemize}

Эти тесты позволяют оценить баланс нулей и единиц, распределение единиц внутри блоков, структуру серий и наличие накопленных отклонений в последовательности.\hypertarget{index_autotoc_md12}{}\doxysection{\texorpdfstring{Описание программы}{Описание программы}}\label{index_autotoc_md12}
Программа состоит из следующих основных частей:\+


\begin{DoxyItemize}
\item реализации трёх пользовательских генераторов;
\item функций генерации выборок случайных чисел;
\item функций вычисления описательных статистик;
\item функций реализации NIST-\/подобных тестов;
\item модуля измерения скорости генерации;
\item блока сохранения результатов в файлы:\+
\begin{DoxyItemize}

\item {\ttfamily output/\+nist\+\_\+results.\+csv};

\end{DoxyItemize}\hypertarget{index_autotoc_md13}{}\doxysection{\texorpdfstring{Результаты экспериментов}{Результаты экспериментов}}\label{index_autotoc_md13}
\hypertarget{index_autotoc_md14}{}\doxysubsection{\texorpdfstring{Описательные статистики}{Описательные статистики}}\label{index_autotoc_md14}
В файл {\ttfamily random\+\_\+stats.\+csv} сохраняются результаты для 20 выборок каждого генератора. По этим данным можно построить графики изменения:\+


\begin{DoxyItemize}

\hypertarget{index_autotoc_md16}{}\doxysubsection{\texorpdfstring{Результаты NIST-\/подобных тестов}{Результаты NIST-\/подобных тестов}}\label{index_autotoc_md16}
Результаты тестов сохраняются в файл {\ttfamily nist\+\_\+results.\+csv}. Для каждого генератора вычисляются p-\/value по всем реализованным тестам. Эти значения позволяют оценить случайность битовой последовательности и выявить возможные статистические отклонения.\hypertarget{index_autotoc_md17}{}\doxysubsection{\texorpdfstring{График NIST-\/подобных тестов}{График NIST-\/подобных тестов}}\label{index_autotoc_md17}
\hypertarget{index_autotoc_md18}{}\doxysubsection{\texorpdfstring{Скорость генерации}{Скорость генерации}}\label{index_autotoc_md18}
Для измерения производительности программа вычисляет время генерации последовательностей длиной от 1000 до 1000000 значений. Результаты сохраняются в файл {\ttfamily speed\+\_\+results.\+csv}.\hypertarget{index_autotoc_md19}{}\doxysubsection{\texorpdfstring{График скорости генерации}{График скорости генерации}}\label{index_autotoc_md19}
\hypertarget{index_autotoc_md20}{}\doxysection{\texorpdfstring{Анализ результатов}{Анализ результатов}}\label{index_autotoc_md20}
По графику средних значений видно, что у всех трёх генераторов среднее по 20 выборкам колеблется около середины диапазона, то есть около 5000, без выраженного систематического смещения в большую или меньшую сторону. Это говорит о том, что на уровне выборочных средних все генераторы в целом дают приемлемое распределение значений в пределах исследуемого диапазона.

График стандартного отклонения показывает, что разброс значений у всех генераторов находится примерно на одном уровне, а колебания от выборки к выборке не выходят за разумные пределы. Коэффициент вариации также остаётся достаточно стабильным:\+ у всех трёх генераторов он лежит примерно в интервале от 0.\+55 до 0.\+61, что указывает на близкую относительную изменчивость выборок.

По критерию хи-\/квадрат большинство выборок проходят проверку на равномерность, поскольку значения статистики обычно находятся ниже критического уровня 30.\+1435. Однако у каждого генератора есть отдельные выборки, в которых наблюдаются отклонения:\+ у \doxylink{class_l_c_g___bays_durham}{LCG\+\_\+\+Bays\+Durham} одна выборка имеет (\textbackslash{}chi\texorpdfstring{$^\wedge$}{\string^}2) =38.\+28, у \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT} одна выборка достигает (\textbackslash{}chi\texorpdfstring{$^\wedge$}{\string^}2) =31.\+12 и ещё одна поднимается до 28.\+96, а у \doxylink{class_additive_l_f_g___rotate_xor}{Additive\+LFG\+\_\+\+Rotate\+Xor} одна выборка даёт (\textbackslash{}chi\texorpdfstring{$^\wedge$}{\string^}2) =30.\+56, то есть слегка превышает критический порог. В целом это означает, что все генераторы в основном демонстрируют равномерность, но полностью идеальным нельзя считать ни один из них.

Результаты NIST-\/подобных тестов показывают, что \doxylink{class_l_c_g___bays_durham}{LCG\+\_\+\+Bays\+Durham} и \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT} проходят все использованные проверки по порогу p-\/value = 0.\+01, поскольку их значения остаются выше этой границы во всех тестах. Наиболее уверенно выглядит \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT}:\+ у него особенно высокое p-\/value в тесте monobit (0.\+212764), тесте runs (0.\+729805) и на обратной кумулятивной сумме p-\/value остаётся комфортно выше порога. У \doxylink{class_l_c_g___bays_durham}{LCG\+\_\+\+Bays\+Durham} также хорошие результаты, причём особенно выделяется тест longest run с p-\/value 0.\+950109.

Наиболее слабые результаты показывает \doxylink{class_additive_l_f_g___rotate_xor}{Additive\+LFG\+\_\+\+Rotate\+Xor}:\+ хотя тесты monobit, block frequency, runs и longest run он проходит, значения p-\/value для cusum\+\_\+forward и cusum\+\_\+backward равны 0.\+009880 и 0.\+007699 соответственно, то есть оказываются ниже контрольного порога 0.\+01. Это указывает на возможные накопленные отклонения в битовой последовательности и делает данный генератор менее устойчивым по сравнению с двумя остальными в рамках проведённых NIST-\/подобных проверок.

По графику скорости генерации видно, что при росте размера выборки время работы всех генераторов увеличивается почти линейно. Самым быстрым на больших объёмах данных оказывается \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT}:\+ при размере 1000000 значений он показывает 4.\+331 мс, тогда как \doxylink{class_l_c_g___bays_durham}{LCG\+\_\+\+Bays\+Durham} требует 4.\+904 мс, \doxylink{class_standard_m_t}{Standard\+MT} — 6.\+493 мс, а \doxylink{class_additive_l_f_g___rotate_xor}{Additive\+LFG\+\_\+\+Rotate\+Xor} — 6.\+214 мс. На промежуточных объёмах \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT} также обычно остаётся среди лидеров, что делает его наиболее выгодным по соотношению качества и производительности.

Таким образом, по совокупности результатов наиболее удачным генератором в данной работе можно считать \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT}, поскольку он показывает хорошие статистические характеристики, проходит все NIST-\/подобные тесты и демонстрирует лучшую скорость генерации на больших выборках. \doxylink{class_l_c_g___bays_durham}{LCG\+\_\+\+Bays\+Durham} тоже даёт в целом качественные результаты, но уступает по скорости и имеет единичный заметный выброс по критерию хи-\/квадрат. \doxylink{class_additive_l_f_g___rotate_xor}{Additive\+LFG\+\_\+\+Rotate\+Xor} показывает приемлемые описательные статистики, однако проигрывает по устойчивости в части NIST-\/подобных тестов и по производительности на больших объёмах генерации.\hypertarget{index_autotoc_md21}{}\doxysection{\texorpdfstring{Выводы}{Выводы}}\label{index_autotoc_md21}
В ходе лабораторной работы были реализованы и исследованы три генератора псевдослучайных чисел:\+ {\ttfamily \doxylink{class_l_c_g___bays_durham}{LCG\+\_\+\+Bays\+Durham}}, {\ttfamily \doxylink{class_xorshift128___m_l_t}{Xorshift128\+\_\+\+MLT}} и {\ttfamily \doxylink{class_additive_l_f_g___rotate_xor}{Additive\+LFG\+\_\+\+Rotate\+Xor}}.

В результате выполнения работы:\+


\begin{DoxyItemize}
\item получены выборки случайных чисел и вычислены их основные статистические характеристики;
\item выполнена проверка равномерности по критерию хи-\/квадрат;
\item проведены NIST-\/подобные тесты для битовых последовательностей;
\item измерена скорость генерации и выполнено сравнение с {\ttfamily std::\+mt19937\+\_\+64}.
\end{DoxyItemize}

Проведённое исследование показывает, что для оценки генераторов необходимо учитывать одновременно два аспекта:\+ статистическое качество последовательности и производительность алгоритма. Наиболее предпочтительным можно считать тот генератор, который показывает устойчивые статистические характеристики, успешно проходит тесты случайности и при этом имеет высокую скорость генерации.\hypertarget{index_autotoc_md22}{}\doxysection{\texorpdfstring{Репозиторий проекта}{Репозиторий проекта}}\label{index_autotoc_md22}
